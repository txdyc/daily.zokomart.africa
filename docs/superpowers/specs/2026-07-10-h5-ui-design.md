# ZokoDaily H5 — UI Design Spec

**Date:** 2026-07-10
**Status:** Approved (mockup reviewed in session)
**Parent spec:** [2026-07-10-zokodaily-news-aggregation-design.md](2026-07-10-zokodaily-news-aggregation-design.md) §7
**Consumed by:** Plan 3 (H5 frontend implementation plan)

This document records the visual decisions approved from the interactive mockup.
Functional behavior (routes, API contracts, pull-to-refresh, share targets) stays in the
parent spec; this file covers only how things look.

## 1. Design tokens

Define as CSS custom properties in `h5/src/styles/tokens.css`. Light mode is the V1
target; keeping all colors behind tokens leaves dark mode as a later drop-in.

```css
:root {
  /* brand — teal ramp, evokes West African palettes */
  --brand-50:  #E1F5EE;   /* light fill, text on dark teal */
  --brand-200: #5DCAA5;   /* icons on dark teal */
  --brand-500: #1D9E75;   /* primary: logo block, active states, links */
  --brand-700: #085041;   /* image-overlay backgrounds, banner scrim */
  --brand-900: #04342C;   /* darkest: toggle pill background */

  /* neutrals */
  --bg:            #FFFFFF;
  --surface:       #F7F6F3;  /* page background behind cards */
  --border:        #E5E3DC;  /* hairline card borders */
  --text-primary:  #1F1F1D;
  --text-secondary:#5F5E5A;  /* ZH translation paragraphs, meta */
  --text-muted:    #93918A;  /* dates, hints */

  /* layout */
  --radius-card: 10px;
  --radius-pill: 999px;
}
```

Country tag colors (badge background/text pairs, used until flag icons render beside them):

| Country | bg | text |
| ------- | -- | ---- |
| GH | #EAF3DE | #3B6D11 |
| NG | #EAF3DE | #3B6D11 |
| SN | #FAEEDA | #854F0B |
| CI | #FCEBEB | #A32D2D |

Note: goal.md requires a **flag icon** next to each country. Render the flag emoji
(🇬🇭 🇳🇬 🇸🇳 🇨🇮, from `country.flag_emoji`) followed by the country name in the current UI
language; the colored two-letter badge from the mockup is the fallback where emoji flags
don't render (some Android WebViews).

## 2. Typography

- Font stack: `system-ui, -apple-system, "PingFang SC", "Noto Sans SC", sans-serif`
  (no webfont download; must render CJK well).
- Two weights only: 400 regular, 500 medium. Headlines and card titles are 500 — never 600/700.
- Sizes: card headline 13px/1.4; detail headline 17px/1.45 (weight 500, the "bold headline"
  from goal.md); body paragraphs 15px/1.65; meta and dates 12px; badges/hints 11px.

## 3. Homepage

- **Header** (sticky, white, hairline bottom border): 26px rounded-square logo block in
  `--brand-500` with white glyph + "ZokoDaily" wordmark (15px, weight 500) on the left;
  on the right a 中/EN pill toggle (hairline border, `--radius-pill`) and a search icon.
- **Banner**: full-width, ~40% of viewport width tall, image with a bottom scrim in
  `--brand-700` at 85% opacity fading upward; headline overlaid in `--brand-50`
  (13–15px, weight 500, max 2 lines). Progress indicator: 5 horizontal dashes
  (3px tall, `--radius-pill`), active dash wider (14px vs 6px) in `--brand-200`,
  inactive in `--brand-700`. Auto-rotates every 5s; swipe to advance; tap navigates
  to the article.
- **News list**: two-column grid, 10px gap, 12px page padding. Card = hairline border,
  `--radius-card`, image on top (16:10, object-fit cover, gray placeholder block with a
  photo icon while loading), then 8px-padded text: headline (max 2 lines, ellipsis),
  then meta row: country flag + name, date (MM-DD). Card headline language follows the
  UI language (ZH title when UI is 中文, source title when EN; fall back to the other
  if missing).
- **Pull-to-refresh**: subtle centered hint row (arrow icon + "下拉刷新" / "Pull to refresh",
  11px `--text-muted`) that appears during over-scroll; infinite scroll appends pages below.

## 4. News detail page

- **Header** (sticky): chevron-left back button; beside it country · category in 12.5px
  `--text-secondary` (e.g. "加纳 · 商业"); share icon on the right opening the share sheet.
- **Main image**: full-width, `--brand-700` placeholder while loading. The content-language
  toggle floats on the image, top-right, 8px inset: a `--brand-900` pill containing three
  segments — `EN｜中｜双语` for English sources, `FR｜中｜双语` for French sources. Active
  segment: `--brand-50` background, `--brand-900` text, weight 500; inactive: transparent,
  `--brand-200` text. Default mode = source language.
- **Meta row** below image: flag + country · category · date (YYYY-MM-DD), 12px muted.
- **Headline**: source-language headline 17px weight 500; when mode is 中 or 双语, show the
  ZH headline (双语 shows both, ZH second in `--text-secondary` at 15px). In source-only
  mode show only the source headline.
- **Source line**: "来源 {site name}" / "Source: {site name}" in 12px `--brand-500`,
  external-link icon, links to `source_url`.
- **Body**:
  - Source-only mode: source paragraphs in `--text-primary`.
  - 中 mode: ZH paragraphs only, styled as primary text.
  - 双语 mode: each source paragraph (primary color) immediately followed by its ZH
    translation styled muted — `--text-secondary`, 2px solid `--border` left rule,
    8px left padding, **no border radius**, 12px bottom margin. This visual pairing is
    the approved treatment for scanning two languages.

## 5. General

- Sentence case everywhere; no all-caps labels.
- Flat surfaces: no gradients, shadows, or glow. Hairline (1px at DPR≥2 via
  `transform: scaleY(0.5)` or 0.5px where supported) borders only.
- All interactive targets ≥ 40×40px even when the glyph is smaller.
- UI strings via vue-i18n (`zh` default, `en` secondary), persisted in localStorage.
